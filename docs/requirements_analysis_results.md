# 요구사항 정의서: 세부 분석 결과 섹션

## 1. 개요
'세부 분석 결과' 섹션은 AI 모델이 사용자의 얼굴을 분석하여 도출한 각 화폐 단위별 확률(가중치)을 시각적으로 보여주는 구역이다. 사용자가 총액이 어떻게 산출되었는지 이해할 수 있도록 상세 데이터를 제공하는 것이 목적이다.

## 2. 주요 기능 및 구성 요소

### 2.1. 결과 요약 영역 (Result Summary)
- **배경 스타일**: 반투명한 파란색-보라색 계열의 배경 박스
  - `background: rgba(99, 102, 241, 0.1)`
  - `border: 1px solid rgba(99, 102, 241, 0.2)`
  - `border-radius: 20px`
- **텍스트 구성**: 
  - 기본 텍스트: "당신은 [금액][단위][권] 입니다."
  - 금액 부분: 주황색/금색 강조 (`color: #fbbf24`)
  - 금액에 텍스트 그림자 효과 적용 (`text-shadow: 0 0 20px rgba(251, 191, 36, 0.4)`)
- **타이포그래피**:
  - 일반 텍스트: `font-size: 2rem`, `font-weight: 800`
  - 금액 부분: `font-size: 2.8rem`

### 2.2. 화폐 비교 이미지 (Comparison Container)
- **화폐 배경 이미지**: 선택된 통화의 지폐 이미지 표시
- **숫자 오버레이**: 
  - 화폐 왼쪽 하단에 계산된 금액 표시
  - 위치: `left: 7%`, `bottom: 8%`
  - 스타일: `font-size: 8.0cqw`, `color: #3d9180`
  - 텍스트 외곽선: `-webkit-text-stroke: 0.3cqw #183722`
- **얼굴 프레임**: 
  - 화폐 오른쪽에 원형으로 사용자 얼굴 표시
  - 위치: `right: 12%`, `top: 50%`
  - 크기: `width: 29.75%`, `aspect-ratio: 1/1.4`
  - 필터 효과: 녹색 톤 세피아 효과

### 2.3. 항목 리스트 (Result Items)
- **섹션 제목**: "세부 분석 결과"
  - 중앙 정렬, 회색 계열 텍스트 (`color: var(--text-muted)`)
- **화폐 레이블**: 해당 화폐의 단위와 통화 명칭을 표시한다.
  - 예: `10,000원`, `1,000엔`
  - i18n 시스템을 통해 국가별 단위(원, 엔 등)가 적용되어야 한다.
  - **주의**: 화폐 아이콘은 표시하지 않음 (레이블만 표시)
- **확률 수치 (Percentage)**: AI 모델이 해당 화폐 단위로 판단한 확률을 표시한다.
  - 소수점 첫째 자리까지 표시 (예: `67.2%`).
  - 레이블 행의 오른쪽 끝에 배치
  - 스타일: `font-weight: 800`, 보라색 계열 (`color: var(--primary)`)
- **프로그레스 바 (Progress Bar)**: 확률 수치를 시각적으로 나타내는 바 형태의 UI.
  - 배경: `rgba(255, 255, 255, 0.05)`, 높이 `12px`, `border-radius: 6px`
  - 게이지: 파란색-보라색 그라디언트 (`linear-gradient(to right, var(--primary), var(--accent))`)
  - 글로우 효과: `box-shadow: 0 0 10px var(--primary-glow)`

### 2.4. 애니메이션 및 인터랙션
- **결과 영역 페이드인**: 전체 결과 영역이 나타날 때 부드러운 페이드인 효과
  - `animation: fadeIn 0.5s ease-out`
- **순차적 등장 (Staggered Animation)**: 결과가 로드될 때 리스트 항목들이 위에서 아래로 순차적으로 나타난다.
  - 각 항목마다 `0.1s` 간격으로 딜레이 적용
  - 애니메이션: `fadeInUp` (투명도 0→1, Y축 이동 20px→0)
  - 지속 시간: `0.5s ease forwards`
- **게이지 채우기 (Fill Animation)**: 항목이 나타난 후 프로그레스 바의 게이지가 0%에서 해당 확률값까지 부드럽게 채워진다.
  - 지속 시간: `1s`
  - 이징 함수: `cubic-bezier(0.17, 0.67, 0.83, 0.67)`
  - 각 항목마다 `100ms` 간격으로 시작
- **스캐닝 효과**: 이미지 업로드 후 미리보기 영역에 스캐닝 라인이 위에서 아래로 이동
  - 높이 `4px`, 파란색-투명 그라디언트
  - `animation: scan 2s linear infinite`

## 3. 비즈니스 로직

### 3.1. 데이터 처리
- **확률 정렬**: 확률이 높은 순서대로 상단에 배치한다.
  - `prediction.sort((a, b) => b.probability - a.probability)`
- **누락 항목 처리**: AI 모델 결과에서 확률이 0%로 판단된 화폐 단위라도 리스트에 포함하여 전체적인 분석 범위를 보여준다.
  - `DENOMINATIONS` 배열의 모든 항목을 확인하여 누락된 클래스를 `probability: 0`으로 추가
- **총액 계산**: 각 화폐 단위의 가치와 확률을 곱한 후 합산
  - `totalValue = Σ(value × probability)`
  - 결과는 반올림하여 정수로 표시
- **단위 변환**: `DENOMINATIONS` 설정에 따라 해당 국가의 모든 화폐 종류가 출력되어야 한다.
- **특수 케이스 처리**: JPY 모델의 경우 'Class 3'을 '10000'으로 매핑

### 3.2. 다국어 지원 (i18n) 요구사항
- **지원 언어**: 한국어(KO) 및 일본어(JA).
- **텍스트 관리**: 모든 UI 텍스트는 `script.js`의 `TRANSLATIONS` 객체에서 관리.
- **언어 전환**: 상단 우측 언어 선택 버튼(KO | JA)을 통해 전환
  - 활성 언어는 보라색(`var(--primary)`)으로 표시
  - 비활성 언어는 회색(`var(--text-muted)`)으로 표시
- **언어별 조합 규칙 (최종 결과 텍스트)**:
  - **한국어 UI**: `당신은 [금액][단위][권] 입니다.` (예: `당신은 17,807원권 입니다.`)
  - **일본어 UI**: `あなたは [금액][단위][券] です。` (예: `あなたは 17,807ウォン券 です。`)
- **금액 표시**: 천 단위 구분 기호(,) 사용
  - JavaScript `toLocaleString()` 메서드 사용
