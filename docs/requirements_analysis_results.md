# 요구사항 정의서: 세부 분석 결과 섹션

## 1. 개요
'세부 분석 결과' 섹션은 AI 모델이 사용자의 얼굴을 분석하여 도출한 각 화폐 단위별 확률(가중치)을 시각적으로 보여주는 구역이다. 사용자가 총액이 어떻게 산출되었는지 이해할 수 있도록 상세 데이터를 제공하는 것이 목적이다.

## 2. 주요 기능 및 구성 요소

### 2.1. 항목 리스트 (Result Items)
- **화폐 아이콘**: 각 화폐 단위를 상징하는 아이콘 배경 이미지를 표시한다.
  - 경로 규칙: `images/item_<prefix>_<value>.png` (예: `item_won_50000.png`)
  - 이미지 부재 시 아이콘 영역을 숨기거나 기본 스타일을 유지한다 (`onerror` 처리).
- **화폐 레이블**: 해당 화폐의 단위와 통화 명칭을 표시한다.
  - 예: `1,000원`, `10,000엔`
  - i18n 시스템을 통해 국가별 단위(원, 엔 등)가 적용되어야 한다.
- **확률 수치 (Percentage)**: AI 모델이 해당 화폐 단위로 판단한 확률을 표시한다.
  - 소수점 첫째 자리까지 표시 (예: `85.5%`).
- **프로그레스 바 (Progress Bar)**: 확률 수치를 시각적으로 나타내는 바 형태의 UI.
  - 배경과 게이지 영역으로 구성된다.

### 2.2. 애니메이션 및 인터랙션
- **순차적 등장 (Staggered Animation)**: 결과가 로드될 때 리스트 항목들이 위에서 아래로 순차적으로 나타난다.
- **게이지 채우기 (Fill Animation)**: 항목이 나타난 후 프로그레스 바의 게이지가 0%에서 해당 확률값까지 부드럽게 채워진다.
- **아이콘 호버 효과**: 각 항목에 마우스를 올리거나 터치 시 화폐 아이콘이 살짝 확대되는 인터랙션을 제공한다.

## 3. 비즈니스 로직

### 3.1. 데이터 처리
- **확률 정렬**: 확률이 높은 순서대로 상단에 배치한다.
- **누락 항목 처리**: AI 모델 결과에서 확률이 0%로 판단된 화폐 단위라도 리스트에 포함하여 전체적인 분석 범위를 보여준다.
- **단위 변환**: `DENOMINATIONS` 설정에 따라 해당 국가의 모든 화폐 종류가 출력되어야 한다.

### 3.3. 다국어 지원 (i18n) 요구사항
- **지원 언어**: 한국어(KO) 및 일본어(JA).
- **텍스트 관리**: 모든 UI 텍스트는 `script.js`의 `TRANSLATIONS` 객체에서 관리.
- **언어별 조합 규칙 (최종 결과 기준)**:
  - **한국어 UI**: `당신은 [금액][단위][권] 입니다.` (예: `50,000원권`, `10,000엔권`)
  - **일본어 UI**: `あなたは [금액][단위][枚/券] です。` (예: `50,000ウォン券`, `10,000円券`)

#### 3.3.1. 대한민국 원화 (KRW) 분석 모드
- **대상**: 100, 1,000, 5,000, 10,000, 50,000.
- **언어 환경별 표기**: 
  - **한국어(KO) UI**: `[금액]원권` (예: `50,000원권`)
  - **일본어(JA) UI**: `[금액]ウォン券` (예: `50,000ウォン券`)
- **이미지**: `item_won_` 접두어 에셋 사용.

#### 3.3.2. 일본 엔화 (JPY) 분석 모드
- **대상**: 1,000, 5,000, 10,000.
- **언어 환경별 표기**: 
  - **한국어(KO) UI**: `[금액]엔권` (예: `10,000엔권`)
  - **일본어(JA) UI**: `[금액]円券` (예: `10,000円券`)
- **특이사항**: AI 모델 클래스 'Class 3'를 '10,000'으로 정확히 매핑하여 처리.
- **이미지**: `item_yen_` 접두어 에셋 사용.

## 4. UI/UX 디자인 요구사항
- **통화별 레이아웃 최적화**: 권종 개수가 다른 경우에도 리스트 정렬이 깨지지 않도록 Flexbox/Grid 레이아웃을 사용한다.
- **디자인 스타일**: 전체 앱의 'Premium Dark Mode'와 'Glassmorphism' 스타일을 유지한다.
- **가독성**: 텍스트와 배경 사이의 충분한 대비를 확보하고, `Outfit` 폰트와 `Roboto Mono` 폰트를 적절히 섞어 데이터의 신뢰감을 준다.
- **응답성 (Responsiveness)**: 모바일 환경에서도 리스트와 프로그레스 바가 깨지지 않고 적절한 비율로 표시되어야 한다.
